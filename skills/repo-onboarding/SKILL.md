# Repository Onboarding

## 触发条件

- 第一次进入仓库
- 切换到不同仓库
- 长时间未处理该项目
- 当前对话与实际代码状态可能不一致
- 用户要求先了解项目整体情况

## 执行步骤

1. 读取 Git 远程地址、默认分支、当前分支和工作区状态。
2. 查找适用的 `AGENTS.md`、README、依赖文件、环境变量示例和测试入口。
3. 检查主要目录、启动入口、配置结构和最近相关提交。
4. 对照 `docs/REPOSITORIES.md` 判断该仓库职责。
5. 搜索当前任务是否已有实现、Issue、TODO、失败测试或相关文档。
6. 列出可验证的任务计划。
7. 判断是否缺少只有用户能提供的信息。

## 信息不足

能从仓库、工具、公开资料和已连接资源中取得的信息自行取得。

真正缺少时主动输出：

- 缺少的信息
- 阻塞原因
- 最快提供方式
- 可以先继续完成的部分

## 输出

保持简短：

- 仓库定位
- 当前状态
- 关键入口
- 任务计划
- 风险与阻塞
