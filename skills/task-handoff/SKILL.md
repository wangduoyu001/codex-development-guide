# Task Handoff

## 触发条件

- 准备切换对话
- 主任务分阶段结束
- 子智能体回传结果
- 需要交给另一个 Agent 继续
- 当前上下文过长或包含过期方案

## 必须记录

1. 任务目标
2. 当前仓库、分支和工作区状态
3. 已完成内容
4. 修改的关键文件
5. 已执行的测试及结果
6. 未完成事项
7. 已知风险、失败尝试和不能重复踩的坑
8. 需要用户提供的信息或授权
9. 下一步最小可执行动作

## 禁止

- 不写“基本完成”而不列未完成项
- 不把未运行的测试写成通过
- 不遗漏未提交修改
- 不隐瞒 Mock、临时方案或硬编码
- 不把推测写成事实

## 推荐格式

```markdown
# 任务交接

## 目标

## 当前仓库与分支

## 已完成

## 修改文件

## 测试结果

## 未完成与风险

## 需要用户提供

## 下一步
```
