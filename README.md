# Codex Development Guide

面向 `wangduoyu001` 多仓库开发环境的 AI 编程助手规范。

这不是一份需要每次完整塞进对话的超长 Prompt。仓库采用三层结构：

1. 根目录 `AGENTS.md`：短入口与导航，仅保留高频硬规则。
2. `docs/`：详细规范，按任务需要读取。
3. `skills/`：可复用、按需加载的操作流程。

## AI 进入任何项目后的默认动作

1. 识别当前仓库、远程地址、分支与工作区状态。
2. 阅读当前仓库适用的 `AGENTS.md`。
3. 搜索现有实现、README、测试与配置。
4. 能自行获得的信息自行获得。
5. 缺少密钥、权限、商业决策或不可逆操作授权时，主动向用户索取，并说明原因与最快提供方式。
6. 完成修改后执行与变更相关的验证，不得把“理论可行”写成“测试通过”。

## 目录

- `AGENTS.md`：AI 入口
- `docs/GLOBAL.md`：全局行为规则
- `docs/REPOSITORIES.md`：仓库地图与职责边界
- `docs/RESEARCH.md`：开源项目与官方资料调研结论
- `skills/repo-onboarding/`：进入仓库并建立上下文
- `skills/task-handoff/`：长任务交接
- `templates/AGENTS.md`：业务仓库模板

## 设计原则

- 根规则保持短小，详细流程按需加载。
- 不把所有规则塞进一个数千行文件。
- 项目自己的 `AGENTS.md` 优先描述真正不可从代码推断的内容。
- AI 不等待用户主动补材料，而是自行检查后主动提出必要信息。
